IO Buster

IO Buster provides a flexible test framework, which allows characterizing performance of an NVMe drive. When combined with IO Buster Coordinator, multiple NVMe drives can be characterized concurrently.

IO Buster leverages the Storage Performance Development Kit (SPDK) to generate high speed IO traffics to NVMe drives.

Supported Platform

IO Buster is designed to work on any Linux distribution with glibc >= 2.7 and kernel version >= 3.16.

We only officially tested IO Buster on the following Linux distributions:

  • Ubuntu 18.04 LTS
  • CentOS 7

Installation

  1. Download IO Buster from Github and extract.

    wget https://github.com/qamaestro/io-buster/releases/latest/download/iob.tgz
tar xzvf iob.tgz

  2. Execute iob install to start installation.

    cd iob-*
./iob install

  3. Configure IO Buster.

    • Before starting the configuration, please ensure NVMe drives are already connected and enumerable.
      • You can confirm if NVMe drives are installed by running lspci | grep "Non-Volatile memory controller".
    • Please ensure Secure Boot is disabled from BIOS.
    iob scan --configure

    Whenever you add additional drives to the host, you need to run this command for proper configuration.

  4. Reboot your host machine to complete configuration.
